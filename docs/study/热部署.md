 热部署与热加载
 ==============================================================================
 >
 ## 联系

 - 无须重启服务器进行编译/部署项目
 - 基于Java的类加载器进行实现

 ## 区别

 - 热部署：在服务器运行时“重新部署项目”
 - 热加载：在运行时重新加载class文件

 ## 原理

 - 热部署直接重新加载整个应用：内存释放，耗费时间长
 - 热加载在运行时重新加载class：容器启动时，启动后台线程，检测类的时间戳

 ## 场景

 - 热部署更多使用在生产环境；
 - 热加载更多使用在开发环境：安全性，无法监控，日志输出

 ## 类加载的五个阶段

 加载：静态存储结构，自定义类加载器
 验证：避免对虚拟的危害，参数设置
 准备：内存布局，赋初始值
 解释：符号引用变成直接引用，调用程序自定义代码
 初始化：有且只有五种情况进行初始化，
1. 关键字， new get static //final
2. 反射
3. 调用子类，父类未初始化
4. 虚拟机启动
5. jdk1.7 动态语言

## 类加载器的特点

- AppClass Loader（系统）
- 类加载器将加载任务交给其父类，如果父类找不到，再由自己加载
- Bootstrap Loader（启动类加载器）是顶级的类加载器，父类为空
